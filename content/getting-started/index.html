Getting Started With Malscan

System Requirements

Malscan has a number of dependencies, many of which are already pre-installed on most minimal Linux installations. Required packages are:
  • ClamAV 0.98+
  • git
  • file
  • bash
In addition to these packages, Malscan requires root SSH or terminal access to the machine that will be running Malscan.

Installation

Automated installation options are available for Red Hat Enterprise Linux and its derivatves as well as Debian and its derivatives.

Red Hat Enterprise Linux / CentOS

Yum repositories are available for the following:
  • Red Hat Enterprise Linux 6.8
  • Red Hat Enterprise Linux 7.2
  • CentOS 6.8
  • CentOS 7.2
To automatically configure the Malscan repository for your RHEL/CentOS installation, run the command below.
[root@example:~]# wget https://www.malscan.org/static/repo-setup.sh | bash

Debian / Ubuntu

An automatic installation script is available for both Ubuntu and Debian, and may function on any other Debian-based operating systems. It has been tested for functionality on the operating systems listed below.
  • Debian 8.2 (Jessie)
  • Debian 7.9 (Wheezy)
  • Ubuntu Server 14.04/li>
To install Malscan using the installation script, run the command below.
[root@example:~]# wget https://www.malscan.org/static/debian-install.sh | bash
The installation script is not unattended. There are a number of prompts that that are required to complete installation. Once installation has been completed, however, the core configuration file in /usr/local/share/malscan/conf.malscan can be included in Puppet or any other configuration management for easier infrastructure-wide installation.
