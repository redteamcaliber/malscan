Contributing to Malscan

Bug Reports, Enhancements and Feature Requests

Application bugs, new feature requests, and improvements can all be submitted through the GitHub Issue Tracker. Please read through each section below before submitting an issue within the issue tracker.

Bug Reports

If you believe you've found with a bug within malscan, please ensure that it is actually within malscan. Issues with ClamAV signature databases or the clamscan or freshclam binaries should be reported to ClamAV instead.
When submitting a bug report on the issue tracker, ensure you have included the following:
  • Your current version of malscan, found using the malscan -v command.
  • The current signature database versions, if applicable, found using the malscan -v command.
  • A full output from the command line of any errors that you have received, and a description of what you were attempting to do.
  • Anything different or unusual with file permissions/ownership/structure that may help pinpoint errors.
All bug reports are confirmed and replicated. If we are unable to replicate your specific error, we may require additional information. Because of this, please ensure that you are actively monitoring your submitted bug report in case additional information is required.

Feature Requests & Enhancements

If you have an idea for a new feature or an enhancement to an existing feature, you can submit a feature request or enhancement issue through the Github Issue Tracker. Always check any outstanding feature and enhancement requests from other users to prevent duplication of the same request. When submitting a feature request or enhancement request, please include as much detail about the feature or enhancement that you'd like to see.
Unfortunately, we cannot always make everyone happy. Many feature requests will remain outside of the scope of Malscan, and as such will be rejected. Please don't take any rejections personally, however we do intend to focus on core functionality and performance improvements whenever required with the limited amount of development resources that are available.

Contributing Code

We are always happy to accept pull requests, provided that they include all of the following:
  • The code is licensed under the MIT license. All Malscan code falls under this license, and yours must not be an exception.
  • Commit messages must be descriptive and follow git best practices for format and information. See this reference.
  • You must be referencing a current issue, whether it is a bug report, feature request, or enhancement.
  • You must update the CHANGELOG.md file under the ### x.x.x-dev section at the top.
  • Your code must have been fully tested on CentOS or RHEL 6 and 7.
Unfortunately, we cannot always make everyone happy. Many feature requests will remain outside of the scope of Malscan, and as such will be rejected. Please don't take any rejections personally, however we do intend to focus on core functionality and performance improvements whenever necessary with the limited amount of development resources that are available.

Contributing Malware Signatures

By default, Malscan does not send any information anywhere. If you choose, however, you can have Malscan send malicious files to the centralized Malscan signatures server for inclusion in future signature databases. As part of this process, the malware is manually reviewed to ensure that it is malicious, and is then added to the malware databases.
Malscan developers do not:
  • Provide investigation or consulting about potentially malicious files for individuals or companies.
  • Provide reports to individuals or companies about any files sent to the Malscan servers
  • Accept any licenses or terms as part of receiving these files
All files that are sent to Malscan through the use of the malscan -r command become the property of Malscan, with permission granted by the sending party to act upon as they wish with no liability or limitations whatsoever.

Donating To Development

Malscan is free and open source software, with no requirements for payment of any type for usage of the software. All development is provided by volunteers who receive no payment for their services by Malscan.
Malscan has very limited resources to pay for things like server hosting and development costs. If you would like to contribute financially to Malscan, you can do so by any of the methods below:
  • Patreon Coming Soon
  • GratiPay Coming Soon
  • Bitcoin
If you would like to donate material services to Malscan, we would be more than happy to accept the following:
  • Server hosting for our web presence or development
  • Mirroring of malware signature databases
If you would like to speak with our development team in regards to donating, you can do so through our Contact Us page.
